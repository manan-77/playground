# Stock Playground

A Robinhood-style investing playground — manage a portfolio, trade, and track
watchlists with **no real money**. Built with React + Vite.

**Prices are live from Yahoo Finance.** The running app fetches quotes and price
history directly from Yahoo's chart API at runtime — there is **no static price
fallback**. Yahoo blocks direct browser calls (CORS), so requests go through the
Vite dev proxy (`/yf` → `query1.finance.yahoo.com`, configured in
`vite.config.js`). The fetch layer is `src/lib/yahoo.js`; `PortfolioContext` is
the single place that calls it.

> Because the proxy only exists in `vite dev` / `vite preview`, the app needs the
> dev server to load market data — a plain static `npm run build` deploy would
> need a real proxy or serverless function in front of Yahoo.

## Run it

```bash
npm install
npm run dev        # http://localhost:5173  (proxies Yahoo for live prices)
```

Other scripts:

```bash
npm run gen-data   # reseed the static ACCOUNT tables (deposits/holdings/etc.)
npm run build      # production build into dist/
npm run preview    # serve the production build (proxy still active)
```

### Optional: AI analysis backend

The stock detail page has an **AI analysis** panel (grounded by a live-API
dossier and an Azure AI Foundry IQ agent). It needs the Python backend running:

```bash
cd backend && python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
uvicorn app:app --reload --port 8000     # Vite proxies /api -> here
```

See `backend/README.md` for the Azure AI Foundry setup. Without it, the panel
shows a hint; without Foundry creds it returns the live dossier + a "configure
Foundry" note.

## What's implemented

Desktop-first, Robinhood-style UI: black theme, lime CTA accent, `#00C805`
gains / `#FF5000` losses, pill buttons. A persistent **top navbar**
(`src/components/TopNav.jsx`) holds every section — Investing, Transactions,
Profile, an inline Search box, and a profile chip.

**Investing / homepage** (`src/pages/HomePage.jsx`)
- Two-column dashboard: a large performance chart + portfolio value on the left,
  a right rail with holdings and watchlist cards.
- Portfolio value header with gain/loss (green up / red down). Hover or drag the
  chart to scrub — the header tracks the cursor.
- Timeframes **1D / 1W / 1M / 1Y / 5Y / ALL**.

**Holdings** (`HoldingsList`, `HoldingRow`)
- Ticker, share count, and a color-coded price button. The price button opens
  **Buy / Sell**; the row opens the stock detail page.

**Buy / Sell flow** (`src/pages/TradePage.jsx`)
- Enter either a **dollar amount** or **share count** (toggle); the other is
  derived from the live price. Updates holdings, average cost, and buying power,
  and records the trade in the activity history.

**Search** (navbar → `SearchModal`)
- Navbar search box opens a centered dialog with live ticker/name suggestions.

**Stock detail** (`src/pages/StockDetailPage.jsx`)
- Live graph + timeframe selector. **Owned** → position details (shares, market
  value, average cost, portfolio diversity, today's/total return). **Unowned** →
  company info, recent news, and analyst buy/hold/sell rating.

**Transactions** (`src/pages/TransactionsPage.jsx`)
- Unified, filterable history (All / Deposits / Buys / Sells) merging bank
  deposits with every buy and sell, plus deposited/invested/proceeds summaries.

**Profile** (`src/pages/ProfilePage.jsx`)
- Editable name, email, and mobile phone, **persisted to localStorage** so edits
  survive a reload. Account-value summary cards at the top.

## Data: live market data + static account tables

**Market data — live from Yahoo, no static copy:**

| Source | Contents |
|--------|----------|
| `src/lib/yahoo.js` | Fetches current price, previous close, full price history per timeframe, 52-week high/low, and volume from Yahoo's chart API (via the `/yf` proxy). |
| `src/data/universe.js` | The tradable ticker list + the **synthetic** fields Yahoo's chart endpoint doesn't return: company description, P/E, analyst rating, news. (Yahoo's other endpoints now need crumb/cookie auth, so these stay synthetic and are labelled in the UI — market cap shows `—`.) |

**Account tables — static JSON, generated by `scripts/generateData.mjs`:**

| File | Spec table | Contents |
|------|-----------|----------|
| `transactions.json` | **Transaction table** | Bank → brokerage deposits: `depositId`, `date`, `fromBank`, `amount`, `status`, `createdAt`. |
| `stockHistory.json` | **Stock history table** | One row per purchase: `transactionId`, `stockId`, `ticker`, `stockPrice`, `purchaseAmount`, `quantityBought`, `date`, `timestamp`. |
| `holdings.json` | derived | Current positions (shares + average cost), derived from the stock-history table. |
| `watchlists.json` | — | Watchlist groups, each with a name and a list of tickers. |

`src/context/PortfolioContext.jsx` fetches a live quote for every universe
ticker on mount, lazily loads each timeframe's history when a chart needs it,
and holds the account state (holdings, cash, watchlists). The account JSON seeds
that state; buys/sells mutate it in memory for the session. Starting buying
power = total deposits − amount spent on seed buys.

## Notes on the spec's open items

- **Market data source:** live from Yahoo Finance at runtime, via the Vite proxy
  (no static fallback). For a deployed static build you'd front Yahoo with a
  proxy/serverless function, or swap in a CORS-friendly provider (Finnhub,
  Alpha Vantage) — only `src/lib/yahoo.js` would change.
- **News API:** news is still synthetic per-stock; swapping in an external news
  API only touches `CompanySection`/`NewsSection` in the detail page.
